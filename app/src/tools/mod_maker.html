Create a regular source engine mod.
A mod should always have client/server .dll either in its "bin" folder or linked via gameinfo from somewhere else, like "episodic/bin".
Linking is handy when you're creating a mod based off Black Mesa, for example.
By default, Source SDK Base 2013 doesn't come with any usable clients with working client/server .dll, so if you're basing your mod off SDK 2013 there are two options:
  • Compile your own client/server .dll
  • Use the precompiled ones from the dropdown in the very bottom of this column.

"Precompiled" doesn't mean anything special at all: They come from public github repository hosted by Valve specifically for the purpose of compiling clients for SDK 2013. (This is literally how a regular sourcemod works)
Linked binaries Cleint/server .dll will be linked via gameinfo instead of copying.
This option will be ignored if you're using precompiled clients for SDk 2013.
Attempt PBR Mount PBR .dll and shaders. The result cannot be predicted, because client could be modified.
Link selected content Link content from selected clients via gameinfo.
Platform should always be selected.

This should only be unchecked if you're absolutely sure you know what you're doing.
In addition, if you're basing your mod off SDK 2013 - ep2, episodic and hl2 should also be selected.
New Mapbase You can try using mapbase with any engine, because why not. It has client/server .dll included so you don't have to worry about anything.

It's still required to link critical content, like platform, hl2, episodic and ep2.

This option will never break anything simply because of the way it works. Meaning that you can always try, see if it works and just forget about it if it doesn't.
PBR Include PBR shader.
